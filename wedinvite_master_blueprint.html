Every API route — method, input, what it does, output, error handling.

POST /api/razorpay/create-order
Triggered byUser clicks "Pay ₹1,299" on Step 4 of order form
Input body{ amount: 129900, currency: "INR", receipt: "order_form_uuid" }
Server doesCalls Razorpay API with RAZORPAY_KEY_SECRET. Creates Razorpay order. Returns orderId to browser.
Output{ razorpay_order_id, amount, currency }
Error handlingRazorpay API down → 503 response → show "Payment service unavailable, try again" toast
POST /api/razorpay/verify
Triggered byRazorpay modal fires onSuccess callback in browser
Input body{ razorpay_payment_id, razorpay_order_id, razorpay_signature, orderFormData }
Server does1. Verify HMAC-SHA256 signature using RAZORPAY_KEY_SECRET. 2. If valid: save order + events + media URLs to Supabase using service role. 3. Generate unique invite_id (nanoid). 4. Set status = "paid".
Output{ success: true, orderId, inviteId } → browser redirects to /order/success
Error handlingSignature mismatch → 400 "Payment verification failed" → show error, DO NOT save order. Duplicate payment_id → 409 → already processed.
POST /api/rsvp/submit
Triggered byGuest submits RSVP form on invite page
Input body{ invite_id, guest_name, attending, guest_count, message }
Server does1. Validate invite_id exists and is not expired. 2. Insert into rsvps table using anon client (RLS allows insert). 3. Return success.
Output{ success: true } → browser shows confetti + thank you message
Error handlingDuplicate RSVP (same name) → 409 → "You've already RSVPed! Thank you." Expired invite → 410 → "This invite is no longer active."
POST /api/upload/media
Triggered byUser drops/selects files in Step 3 of order form
InputFormData: { file, media_type, order_temp_id }
Server does1. Validate file type and size. 2. Generate unique filename (nanoid + extension). 3. Upload to Supabase Storage bucket "invite-media" using service role. 4. Return public URL.
File limitsPhotos: JPG/PNG/WEBP max 5MB. Video: MP4/MOV max 100MB. Audio: MP3/M4A max 10MB.
Error handlingFile too large → 413 → "File too large. Max 5MB for photos." Wrong type → 415 → "Only JPG/PNG allowed for photos."
POST /api/admin/deliver
Triggered byYou click "Mark as live" on admin order detail page
Input body{ order_id } (admin cookie verified by middleware)
Server does1. Verify admin cookie. 2. Set order status = "live". 3. Set delivered_at = now(). 4. Set expires_at = now() + 60 days. 5. Return invite URL.
Output{ invite_url: "https://wedinvite.in/invite/abc123" } → admin copies and sends via WhatsApp

Row Level Security policies
orders — INSERTAllow all (anon). Anyone can create a pending order during checkout flow.
orders — SELECTAnon can only read their own invite row (WHERE invite_id = input AND status = 'live'). Service role reads all.
orders — UPDATEService role only. No client-side updates ever.
events — SELECTAllow all. Events are public — needed to render invite page.
media — SELECTAllow all. Media URLs are public — needed to render invite page.
media — INSERT/UPDATEService role only. Admin uploads final assets.
rsvps — INSERTAllow all (anon). Any guest can RSVP.
rsvps — SELECTService role only. Only admin can view RSVP list.

Music player — technical detail
Browser blocks audio autoplay until user gesture. Solution: add a "Tap anywhere to begin" overlay on first load. On first tap: dismiss overlay + start music. Store user preference in sessionStorage so it doesn't re-show on page refresh during same session.
Invite expiry check
In app/invite/[id]/page.tsx server component: fetch order, check expires_at. If null (not yet delivered) → show "Coming soon" page. If expired → show expired page with renewal CTA. If live → render invite. This check is server-side — no flicker.

Every security measure — production grade, nothing skipped.

Payment verification
Never trust the browser for payment success. Always verify Razorpay HMAC-SHA256 signature server-side before saving any order. A forged request without the valid signature will be rejected with 400.
Admin protection
middleware.ts intercepts every /admin/* request. Checks for httpOnly cookie set at login. No cookie or wrong value → redirect to /admin/login. Cookie is httpOnly so JS cannot read it.
Service role key
SUPABASE_SERVICE_ROLE_KEY only used in server API routes. Never passed to browser. Never in any NEXT_PUBLIC_ variable. Bypasses RLS — treat like a root password.
File upload validation
Server validates file type by checking magic bytes (not just extension). Validates file size before accepting upload. Sanitises filename — strip special characters, use nanoid instead.
RLS on Supabase
Even if someone finds your anon key, RLS prevents them reading other users' orders, accessing admin data, or modifying any record. Each table has explicit policies.
Rate limiting
Vercel's built-in rate limiting on API routes. Add extra protection on /api/rsvp/submit (max 3 per IP per invite) and /api/razorpay/create-order (max 5 per IP per hour).
CORS
API routes only accept requests from NEXT_PUBLIC_APP_URL. Set in next.config.ts. Prevents other sites from calling your payment APIs.
Input sanitisation
All text inputs from order form are sanitised before Supabase insert. Strip HTML tags, limit lengths (names max 100 chars, messages max 500 chars). Prevents XSS in invite pages.

Every performance decision — invite pages must load fast on slow Indian mobile networks.

Image optimisation
Use Next.js <Image> component for all photos. Auto-converts to WebP. Lazy loads below-fold images. Set sizes prop correctly. Target: each photo under 200kb after compression.
Video — don't autoload
Never autoload video on mobile. Show poster image with play button. Only load video when user taps play. Cinematic template: load video only after hero animation completes.
Music — defer
Don't load audio file until first user tap. Use HTML5 Audio API with preload="none". This prevents the audio file from consuming bandwidth before the user wants it.
Font loading
Load Google Fonts with display=swap. Only load the 2 weights you use (400 + 600). Subset to Latin + Devanagari/Tamil only where needed. Preconnect to fonts.googleapis.com.
GSAP — load once
Import GSAP at root layout level — loaded once, cached across pages. Don't import per-component. Use @gsap/react useGSAP hook for cleanup.
Supabase query
In invite page: single JOIN query fetches order + events + media in one round trip. Don't make 3 separate queries. Use Supabase select with nested relation syntax.
Static pages
Homepage, /templates, /pricing, /how-it-works are all server components with no dynamic data — statically generated at build time. Instant load from Vercel CDN edge.
Core Web Vitals targets
LCP (Largest Contentful Paint): under 2.5s on 4G. CLS (Cumulative Layout Shift): under 0.1 — reserve image dimensions. FID/INP: under 100ms — no heavy JS on main thread during load.
