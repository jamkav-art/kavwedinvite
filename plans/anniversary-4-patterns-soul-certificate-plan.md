# Anniversary Quiz — Complete Engagement & Soul Certificate Master Plan

## Core Philosophy

> **"Not a quiz. An experience that couples will remember forever and feel compelled to share."**

Every interaction point must evoke an emotion: **curiosity → nostalgia → laughter → intimacy → awe → pride → desire to share.**

## 4. Advanced Scoring Algorithm

### 4.1 Per-Pattern Scoring

Each pattern has **5 questions**, each scored 0-100:

- **Correct answer:** 100 points
- **Close answer (2nd closest):** 50 points
- **Wrong answer:** 0 points

Pattern Score = (Sum of question scores) / (Number of questions in pattern) × 100

### 4.2 Soul % Calculation

```
Soul % = (Pattern 1% × 0.25) + (Pattern 2% × 0.25) + (Pattern 3% × 0.30) + (Pattern 4% × 0.20)
```

- **Pattern 3 (Soul Layers)** gets highest weight (30%) — deep connection matters most
- **Pattern 2 (LOL Challenge)** gets lowest weight (20%) — fun but less important

### 4.3 Love Language Detection

Based on Pattern 3 (Soul Layers) question about love language — displayed independently on certificate.

### 4.4 Soul Level (Depth Score)

Based on Pattern 3 answers:

- 8-10/10 correct on deep questions → "Soulmates"
- 5-7/10 → "Growing Together"
- 0-4/10 → "Room to Grow" (gentle, encouraging)

### 4.5 Combined Challenge Score (Ultimate Soul Certificate)

When both partners have taken each other's quizzes:

```
Ultimate Soul % = (Partner A Score × 0.5) + (Partner B Score × 0.5)
```

- If Ultimate Soul % > 85 → "Cosmic Connection ✨"
- If 65-85 → "Power Couple 💪"
- If 50-65 → "Beautiful Journey 🌱"
- If < 50 → "Adventure Awaits 🗺️"
- **Bonus:** Show comparison radar chart of both partners' strengths

### 7.2 Database Schema Additions

```sql
-- Extend orders table or create new anniversary_quizzes table
CREATE TABLE anniversary_quizzes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  invite_id TEXT UNIQUE NOT NULL,            -- nanoid for URL
  partner_a_name TEXT NOT NULL,
  partner_b_name TEXT NOT NULL,
  anniversary_date DATE NOT NULL,
  years_together INT,
  couple_photo_url TEXT,
  love_note TEXT,
  pattern_selections JSONB NOT NULL,         -- {"golden-memories": [0,1,2], "lol-challenge": [3,4], ...}
  questions JSONB NOT NULL,                  -- Full question objects selected
  floral_theme TEXT DEFAULT 'rose',
  color_mood TEXT DEFAULT 'romantic-pink',
  background_music TEXT,
  payment_id TEXT,
  paid BOOLEAN DEFAULT FALSE,
  challenger_quiz_id UUID REFERENCES anniversary_quizzes(id),  -- Viral loop
  combined_score NUMERIC(5,2),               -- Ultimate Soul % after challenge
  created_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ
);

CREATE TABLE quiz_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quiz_id UUID NOT NULL REFERENCES anniversary_quizzes(id),
  taker_name TEXT DEFAULT 'Partner B',
  answers JSONB NOT NULL,                    -- [{questionIndex: 0, selectedOption: 1}, ...]
  scores JSONB NOT NULL,                     -- {golden-memories: 80, lol-challenge: 66, ...}
  soul_percentage NUMERIC(5,2) NOT NULL,
  love_language TEXT,
  soul_level TEXT,
  is_challenge BOOLEAN DEFAULT FALSE,        -- Is this a challenge response?
  challenge_completed BOOLEAN DEFAULT FALSE,
  session_token TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 11. Key Risks & Mitigations

| Risk                                        | Probability | Impact | Mitigation                                                |
| ------------------------------------------- | ----------- | ------ | --------------------------------------------------------- |
| Quiz page performance (images + animations) | Medium      | High   | Lazy loading, GPU-accelerated CSS, code splitting         |
| Challenge loop infinite regress             | Low         | Medium | Limit to max 1 challenge level                            |
| Image export quality                        | Medium      | Medium | Use high-DPI canvas, test on mobile Safari                |
| Payment race condition                      | Low         | High   | Server-side verification webhook (existing)               |
| User abandons mid-wizard                    | Medium      | Medium | Zustand persistence saves progress; resume where left off |
| Quiz data too large for localStorage        | Low         | Low    | Store minimal state; fetch full data from DB post-payment |

---

## 12. Final Recommendation

Build this in **Phase 1 + Phase 2 first** (Wizard + Quiz Experience) to validate the core UX, then add **Phase 3-4** (Soul Certificate + Challenge Loop) once the foundation is solid. This delivers value incrementally and allows user testing before investing in the viral loop.

**The 4 quiz patterns are the secret sauce** — they transform a simple quiz into an emotional journey that feels personal, fun, and deep. Combined with the Soul Certificate visual payoff and the challenge loop, this creates a product that users _want_ to share.
