Every component — what it does, what props it takes, which library animates it.

Invite page components
InviteHero.tsxFull-screen opening. Couple names with GSAP text split reveal. Background: photo / color / looping video. Template-specific font + colors loaded from config.
PhotoGallery.tsxSwipeable carousel (use Embla or Swiper). Framer Motion scale-in on scroll. Lightbox on tap. Props: photos[] (array of URLs), coupleName.
MusicPlayer.tsxFloating bottom-right button. Plays MP3 on first user tap. Animated music bars icon when playing. useAudioPlayer hook. Props: audioUrl, autoPlay.
VoiceMessage.tsxA card with "Hear from us" heading. Play/pause button. Progress bar. Framer Motion fade-in. Props: voiceUrl, coupleName.
VideoSection.tsxInline video player. Muted autoplay option. Poster image. Full-screen toggle. Cinematic template uses this as page background instead.
CountdownTimer.tsxDays/Hours/Mins/Secs boxes. useCountdown hook updates every second. GSAP flip animation on each digit change. On expiry: shows "We are married!" + Confetti.
EventDetails.tsxOne card per event (Wedding, Mehndi, Reception). Date, time, venue. Add-to-calendar button. Framer Motion staggered card entrance. Props: events[].
VenueMap.tsxGoogle Maps iframe embed. "Get directions" button (opens maps app). Props: lat, lng, venueName, address.
RSVPForm.tsxGuest name, attending toggle, guest count, blessing message. POST to /api/rsvp/submit. Success: Framer Motion confetti burst + thank you message.
ShareButtons.tsxWhatsApp native share, Copy link (clipboard API), "Download for Instagram story" (html2canvas). Props: inviteUrl, coupleName.
Home page components
HeroSection.tsxFull-screen hero. GSAP SplitText on headline. Floating sample invite card. Two CTAs: "See templates" + "Order now — ₹1,299". Petal/confetti particles.
TemplatePreviewGrid.tsxGrid of 8 template cards. Each has thumbnail, name, color swatches, "Live demo" button. Framer Motion hover lift effect. Reads from lib/templates.ts config.
HowItWorks.tsx4 steps with icons. GSAP ScrollTrigger draws a connecting line between steps as you scroll. Each step fades in on reach.
Testimonials.tsxAuto-scrolling horizontal ticker of testimonials. Framer Motion infinite x animation. 5-star rating display. Couple name + city.
Order form components
OrderStepper.tsxTop progress bar — 4 steps. Active step highlighted. Framer Motion slide transition between steps. State managed with useReducer.
Step3Media.tsxDrag-and-drop photo upload (react-dropzone). Video upload with size limit check. Audio recorder OR file upload toggle. Preview before submit.
Step4Payment.tsxOrder summary card. "Pay ₹1,299" button → calls /api/razorpay/create-order → opens Razorpay modal → on success calls /api/razorpay/verify → redirect to /order/success.

Exactly where to use GSAP vs Framer Motion — and what animation to build in each place.

Rule of thumb
Use GSAP for
Scroll-triggered animations (ScrollTrigger). Text split/reveal effects. Timeline sequences (multiple elements). Canvas / SVG path animations. Digit flip (countdown). Heavy orchestration.
Use Framer Motion for
Component mount/unmount (AnimatePresence). Hover and tap interactions. Page transitions. Staggered list reveals. Spring physics (card lift, modal open). React state-driven animations.
Animation map — every effect planned
Homepage hero — headline reveal
GSAP
SplitText splits headline into chars. gsap.from with stagger 0.03, y:60, opacity:0, duration:0.8. Runs on page load after 300ms delay.
How it works — step line draw
GSAP
SVG connecting line between 4 steps. ScrollTrigger scrubs drawSVG from 0% to 100% as user scrolls through section. Each step icon pops in when line reaches it.
Countdown timer — digit flip
GSAP
On each second tick, gsap.fromTo on the digit: rotateX from -90 to 0, duration 0.3. Gives a mechanical flip clock feel. Only runs on changed digits.
Invite hero — couple name entrance
GSAP
GSAP timeline: first name slides from left, "&" fades in, last name slides from right. 1.2s total. Plays once on page load.
Petal / confetti rain
GSAP
20–30 div elements with petal emoji/shape. Each gets random x, rotation, duration 3–6s. gsap.to with repeat:-1, yoyo on opacity. Runs for 4s then stops.
Template card hover
Framer
whileHover: scale 1.03, y -4. whileTap: scale 0.98. transition: spring, stiffness 300. Applied to each template card wrapper.
Page transitions
Framer
AnimatePresence wraps page content in layout.tsx. Each page: initial opacity:0 y:20 → animate opacity:1 y:0 → exit opacity:0. 0.3s ease.
Order form step transitions
Framer
AnimatePresence mode="wait". Each step slides out left, new step slides in from right. Progress bar width animates with Framer Motion layout prop.
Invite sections scroll reveal
Framer
useInView hook on each section. whileInView: opacity 1 y 0. initial: opacity 0 y 40. viewport: once:true. Stagger children with variants.
RSVP success burst
Framer
On RSVP submit success: AnimatePresence shows a confetti component. 12 colored dots animate outward from center with spring physics. Disappears after 2s.
Testimonial ticker
Framer
Two identical rows of testimonials. Framer motion.div x: 0 to -50% with repeat:Infinity, duration:25, ease:linear. Pauses on hover.
Music player icon bars
Tailwind
3 bars in the music button. Tailwind animate-bounce with different delays (delay-75, delay-150). Shows when isPlaying === true.

Full Supabase schema — every table, column, type, and relation.

orders table
id uuid PRIMARY KEY DEFAULT gen_random_uuid() created_at timestamptz DEFAULT now() couple_name_1 text NOT NULL ← bride name couple_name_2 text NOT NULL ← groom name wedding_date date NOT NULL template_slug text NOT NULL ← e.g. 'royal-maroon' phone text NOT NULL email text NOT NULL special_message text status text DEFAULT 'pending' ← pending/building/live invite_id text UNIQUE ← public URL slug payment_id text ← Razorpay payment ID amount_paid integer ← in paise (129900) delivered_at timestamptz
events table
id uuid PRIMARY KEY DEFAULT gen_random_uuid() order_id uuid REFERENCES orders(id) ON DELETE CASCADE event_name text NOT NULL ← 'Wedding', 'Mehndi' etc event_date date NOT NULL event_time text ← '7:00 PM' venue_name text NOT NULL venue_address text NOT NULL venue_lat numeric venue_lng numeric dress_code text sort_order integer DEFAULT 0
media table
id uuid PRIMARY KEY DEFAULT gen_random_uuid() order_id uuid REFERENCES orders(id) ON DELETE CASCADE media_type text NOT NULL ← 'photo','video','voice','song' file_url text NOT NULL ← Supabase Storage public URL sort_order integer DEFAULT 0
rsvps table
id uuid PRIMARY KEY DEFAULT gen_random_uuid() created_at timestamptz DEFAULT now() invite_id text NOT NULL ← matches orders.invite_id guest_name text NOT NULL attending boolean NOT NULL guest_count integer DEFAULT 1 message text
Supabase Storage buckets
invite-photos
Public bucket. Accepts JPG/PNG up to 5MB each. Max 10 per order.
invite-videos
Public bucket. MP4/MOV up to 100MB. Compress before upload.
invite-audio
Public bucket. MP3/M4A up to 10MB. Voice message + background song.
Row Level Security (RLS) rules
ordersINSERT: allow all (anyone can place order). SELECT: only authenticated admin can read all. No UPDATE/DELETE from client.
rsvpsINSERT: allow all (guests submit RSVP). SELECT: admin only. No UPDATE/DELETE.
mediaSELECT: allow all (public invite pages need to read). INSERT/UPDATE: admin only (you upload final assets).
Storage bucketsAll 3 buckets: public read, authenticated write only. Set in Supabase dashboard.

Run these steps in order — from zero to deployed in one session.

1
Create GitHub repo
New repo: wedinvite. Clone locally. This is your source of truth.
2
Scaffold Next.js project
npx create-next-app@latest wedinvite \
--typescript --tailwind --app --src-dir=false
cd wedinvite
3
Install all dependencies
npm install @supabase/supabase-js @supabase/ssr \
framer-motion gsap @gsap/react \
razorpay react-dropzone \
embla-carousel-react \
react-hook-form zod @hookform/resolvers \
resend date-fns
4
Create Supabase project
Go to supabase.com → New project → Copy Project URL and anon key + service role key. Run the SQL migration to create all 4 tables. Create 3 storage buckets.
5
Create Razorpay account
razorpay.com → Sign up → Get Test Key ID + Secret. Test mode first, switch to live after testing.
6
Set up .env.local
Add all 7 environment variables (see Env Vars tab). Never commit this file — it's in .gitignore by default.
7
Connect to Vercel
vercel.com → Import GitHub repo → Add all env vars in Vercel dashboard → Deploy. Every git push auto-deploys.
8
Connect domain
Buy wedinvite.in on GoDaddy/Namecheap (~₹800/year). Add to Vercel → Domains. Point nameservers. SSL auto-configured.
9
Build in this order
1. lib/templates.ts config → 2. One demo invite HTML → 3. Homepage → 4. Order form → 5. API routes → 6. Admin dashboard → 7. Remaining 7 templates
10
Test end-to-end
Place a test order → pay with Razorpay test card (4111 1111 1111 1111) → verify order appears in Supabase → manually set live → open invite URL → submit RSVP → check Supabase.

Every environment variable needed — copy this into your .env.local file.

# Supabase NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci... SUPABASE_SERVICE_ROLE_KEY=eyJhbGci... ← server only, never public
# Razorpay NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_test_xxxx RAZORPAY_KEY_SECRET=xxxx ← server only
# Email (Resend) RESEND_API_KEY=re_xxxx
# App NEXT_PUBLIC_APP_URL=https://wedinvite.in ADMIN_PASSWORD=your_strong_password ← for admin dashboard
Vercel env var setup
In Vercel dashboard → Project → Settings → Environment Variables. Add all variables above. For production, replace Razorpay test keys with live keys. SUPABASE_SERVICE_ROLE_KEY and RAZORPAY_KEY_SECRET must NEVER be prefixed with NEXT_PUBLIC_ — they are server-only secrets and will be exposed to the browser if you do.
Supabase client setup — lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr'
export const supabase = createBrowserClient(
process.env.NEXT_PUBLIC_SUPABASE_URL!,
process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!

Supabase server setup — lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
export async function createClient() {
const cookieStore = await cookies()
return createServerClient(
process.env.NEXT_PUBLIC_SUPABASE_URL!,
process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
{ cookies: { getAll() { return cookieStore.getAll() } } }
)
}
