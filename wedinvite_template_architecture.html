The secret: you only build ONE invite page. All 8 templates are just different CSS configs injected into the same React component tree. This means fixing a bug fixes it for all 8 templates instantly.
Wrong way to build templates
8 separate HTML files, each with its own layout. When you want to add a feature (e.g. voice message), you have to add it 8 times. Bugs multiply. Maintenance is a nightmare.
Right way — config-driven
One app/invite/[id]/page.tsx that reads a template config object. The config holds colors, fonts, layout variant, animation style, border art. Swap the config → completely different look.
What actually changes between templates
Colors
Background, primary, accent, text — 4 color variables per template
Fonts
Heading font (serif/script/sans) + body font — loaded from Google Fonts
Layout
Hero style: fullscreen photo / color bg / video bg / split layout
Border art
SVG decorative borders — floral, kolam, minimal line, none

What stays the same across all 8
CountdownTimer · PhotoGallery · MusicPlayer · VoiceMessage · VideoSection · EventDetails · VenueMap · RSVPForm · ShareButtons · CoupleStory — all identical components, all receive the same props. Only the visual styling wrapping them changes.

Each template is just a TypeScript object. Here's the full shape.

export interface TemplateConfig {
  slug: string // 'royal-maroon'
  name: string // 'Royal Maroon'
  category: string // 'traditional' | 'modern' | 'regional'
  heroStyle: HeroStyle // 'fullscreen-photo' | 'color-bg' | 'video-bg' | 'split'
  sectionOrder: string[] // ['hero','gallery','countdown','events','map','rsvp']
  colors: {
    bg: string // '#3D0C0C'
    primary: string // '#C9A84C'
    accent: string // '#7B1C1C'
    text: string // '#FDF6E3'
    cardBg: string // 'rgba(255,255,255,0.07)'
    border: string // 'rgba(201,168,76,0.3)'
  }
  fonts: {
    heading: string // 'Playfair Display'
    body: string // 'Lato'
    googleFontsUrl: string // full Google Fonts import URL
  }
  decorations: {
    borderArt: string | null // 'mandap' | 'floral' | 'kolam' | 'mehndi' | null
    divider: string // 'ornamental' | 'floral' | 'line' | 'dots'
    animation: string // 'petals' | 'stars' | 'leaves' | 'fade' | 'confetti'
    cardStyle: string // 'flat' | 'bordered' | 'outlined-gold' | 'glass'
  }
  features: {
    multiEvent: boolean // Festive Mehndi = true
    videoHero: boolean // Cinematic = true
    regionalScript: boolean // South Indian = true
    coupleStory: boolean // optional section
  }
}
How the page uses it
// app/invite/[id]/page.tsx
const config = getTemplateConfig(order.template_slug)

// Inject CSS variables into the page
const cssVars = {
  '--t-bg': config.colors.bg,
  '--t-primary': config.colors.primary,
  '--t-accent': config.colors.accent,
  '--t-heading-font': `'${config.fonts.heading}'`,
} as React.CSSProperties

return (
  <div style={cssVars}>
    <InviteLayout config={config} data={inviteData} />
  </div>
)

What your clients can customise inside their chosen template — and how you implement it.

Level 1 — always included (order form fields)
coupleNamesBride & groom names. Displayed in hero in template's heading font. Client fills in order form step 2.
weddingDatePowers the live countdown timer. Stored as ISO date string in Supabase.
photos[]Up to 10 photos. Uploaded in step 3. Stored in Supabase Storage. Displayed in gallery carousel.
backgroundSongMP3 upload or YouTube link. Auto-plays on page load after first user tap.
voiceMessageMP3 of bride/groom speaking. Plays when guest taps the voice message card.
videoCouple reel / pre-wedding video. Displayed in video section or as hero background (Cinematic template).
events[]Array of events: name, date, time, venue, address. Each becomes an event card with map.
specialMessageA personal message from the couple displayed in a quote card near the bottom.

SVG border art — build once, reuse across templates
You only need 4 SVG border art files. Each is a repeating pattern tile that you apply to the top and bottom of the invite page using CSS background-repeat. Build them once as SVG files in public/borders/ and reference from template config. Claude can generate all 4 for you.
